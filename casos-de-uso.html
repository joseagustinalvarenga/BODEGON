Casos de Uso
Club el Bodegón

Sistema de fidelización y gestión de miembros
Versión 1.0  ·  Marzo 2026
ACTORES

Usuarios del sistema

Visitante

Persona no registrada que accede al sitio. Solo puede ver el catálogo público y registrarse.

Miembro

Cliente registrado del club. Acumula puntos, consulta su perfil y canjea recompensas.

Administrador

Staff del Bodegón con acceso completo al panel de gestión. Gestiona miembros, puntos, recompensas y canjes.

MÓDULO 1

Autenticación

CU-01 Registrarse en el club
ActorVisitante
PrecondiciónNo tener cuenta registrada
Flujo principal
  1. El visitante ingresa nombre completo, email y contraseña
  2. El sistema crea la cuenta con rol MIEMBRO
  3. Se genera automáticamente el perfil de miembro con 0 puntos y nivel BRONCE
  4. Recibe un token JWT y queda autenticado
ResultadoAcceso al panel de miembro
CU-02 Iniciar sesión
ActorVisitante (ya registrado)
PrecondiciónTener cuenta activa
Flujo principal
  1. Ingresa email y contraseña
  2. El sistema valida las credenciales
  3. Se actualiza la fecha del último acceso
  4. Recibe un token JWT
ResultadoAcceso al panel según su rol (Miembro o Admin)
Flujo alternativo: Si las credenciales son incorrectas → error 401 Unauthorized
MÓDULO 2

Perfil del Miembro

CU-03 Ver mi perfil
ActorMiembro
Información visible Nombre completo, email, teléfono, nivel (Bronce / Plata / Oro), puntos disponibles, puntos totales acumulados, cantidad de visitas y fecha de alta en el club
CU-04 Actualizar datos personales
ActorMiembro
Flujo principal
  1. Accede a su perfil
  2. Modifica nombre, teléfono o fecha de nacimiento
  3. Guarda los cambios
RestricciónNo puede modificar su email ni su rol
CU-05 Cambiar contraseña
ActorMiembro
Flujo principal
  1. Accede a la sección "Cuenta"
  2. Ingresa su contraseña actual (verificación obligatoria)
  3. Ingresa la nueva contraseña
  4. El sistema actualiza la contraseña cifrada
Flujo alternativo: Si la contraseña actual es incorrecta → error 401 Unauthorized
MÓDULO 3

Puntos y Niveles

CU-06 Ver historial de transacciones
ActorMiembro
Flujo principal
  1. Accede a "Transacciones"
  2. El sistema muestra la lista paginada de movimientos con tipo, origen, puntos, descripción y fecha

Tipos de transacción

TipoOrigenDescripción
EARNPURCHASEPuntos por compra en el local
EARNVISITPuntos por visita registrada
EARNBIRTHDAYBonus de cumpleaños
EARNREFERRALPuntos por referido
REDEEMREWARD_REDEEMPuntos gastados al canjear una recompensa
ADJUSTADMIN_ADJUSTAjuste manual realizado por el administrador
CU-07 Consultar mi nivel
ActorMiembro
Descripción El nivel se calcula automáticamente en base a los puntos totales acumulados históricamente (no el saldo actual disponible). Sube automáticamente al superar cada umbral.
NivelCondición
BronceNivel inicial de todo miembro registrado
PlataSupera el umbral de plata (configurable por el admin)
OroSupera el umbral de oro (configurable por el admin)
El nivel nunca retrocede aunque el miembro canjee puntos y baje su saldo.
MÓDULO 4

Recompensas

CU-08 Ver catálogo de recompensas
ActorVisitante / Miembro
Flujo principal
  1. Accede a la sección "Recompensas"
  2. El sistema muestra todas las recompensas activas con nombre, descripción y costo en puntos
CU-09 Canjear una recompensa
ActorMiembro
PrecondiciónTener saldo de puntos suficiente. La recompensa debe estar activa y con stock disponible.
Flujo principal
  1. El miembro selecciona una recompensa del catálogo
  2. El sistema verifica que tenga puntos suficientes
  3. Se descuentan los puntos del saldo actual del miembro
  4. Se registra la transacción como REDEEM
  5. Se genera un código único con formato BDG-XXXXXX
  6. El código tiene vigencia de 7 días (configurable)
  7. El código queda en estado ISSUED esperando validación del admin
Flujo alternativo: Si no tiene puntos suficientes → error 400 Bad Request
CU-10 Ver mis canjes
ActorMiembro
Flujo principal
  1. Accede a "Mis Canjes"
  2. Ve el historial con: nombre de la recompensa, código, puntos gastados, fecha y estado actual

Estados de un canje

EstadoSignificado
ISSUEDCódigo generado, esperando validación del admin en el local
REDEEMEDValidado por el admin — recompensa entregada
EXPIREDEl código venció sin ser presentado al admin
CANCELLEDAnulado manualmente por el admin
MÓDULO 5

Promociones

CU-11 Ver promociones públicas
ActorVisitante / Miembro
Flujo principal Accede a la sección de promociones y ve las promociones de tipo PUBLIC que estén vigentes (entre su fecha de inicio y fin)
CU-12 Ver promociones exclusivas para miembros
ActorMiembro
Flujo principal Al acceder con sesión iniciada, el miembro ve tanto las promociones PUBLIC como las MEMBERS_ONLY que estén vigentes
RestricciónLas promociones MEMBERS_ONLY no son visibles sin iniciar sesión
MÓDULO 6

Admin — Gestión de Miembros

CU-13 Ver lista de todos los miembros
ActorAdministrador
Información visible Nombre, email, nivel, puntos actuales, puntos totales acumulados, cantidad de visitas y fecha de alta en el club de cada miembro registrado
CU-14 Sumar puntos a un miembro
ActorAdministrador
Flujo principal
  1. Selecciona un miembro del listado
  2. Ingresa la cantidad de puntos a sumar y una descripción del motivo
  3. El sistema registra la transacción (tipo ADJUST, origen ADMIN_ADJUST)
  4. Se actualiza el saldo del miembro y se recalcula su nivel si corresponde
CU-15 Restar puntos a un miembro
ActorAdministrador
Flujo principal
  1. Selecciona un miembro del listado
  2. Ingresa un valor negativo de puntos y una descripción
  3. El sistema descuenta los puntos del saldo y registra la transacción
PrecondiciónEl miembro debe tener saldo suficiente para la deducción
CU-16 Ver estadísticas generales
ActorAdministrador
Información visible Total de miembros registrados, desglose por nivel (Bronce / Plata / Oro), total de puntos emitidos históricamente y total de puntos en circulación (saldos actuales de todos los miembros)
MÓDULO 7

Admin — Gestión de Recompensas

CU-17 Crear una recompensa
ActorAdministrador
Flujo principal
  1. Accede a "Recompensas" en el panel admin
  2. Completa el formulario: nombre (obligatorio), descripción, costo en puntos (obligatorio, mínimo 1), stock (vacío = ilimitado) y fechas de validez opcionales
  3. La recompensa se crea en estado activa por defecto
Validación: El nombre es obligatorio. Sin nombre → error 400 Bad Request
CU-18 Editar una recompensa
ActorAdministrador
Flujo principal
  1. Selecciona la recompensa a editar desde el listado completo
  2. Modifica los campos deseados (nombre, descripción, costo, stock, fechas)
  3. Guarda los cambios
CU-19 Activar / Desactivar una recompensa
ActorAdministrador
Flujo principal
  1. En el listado de recompensas, alterna el estado con el botón de activar/desactivar
  2. El estado cambia de activa a inactiva o viceversa
EfectoUna recompensa inactiva no aparece en el catálogo del miembro y no puede ser canjeada
MÓDULO 8

Admin — Validación de Canjes

CU-20 Ver canjes pendientes de validación
ActorAdministrador
Información visible Lista de todos los códigos en estado ISSUED con: código BDG-XXXXXX, nombre del miembro, recompensa correspondiente, puntos gastados y fecha de vencimiento
CU-21 Validar un código de canje
ActorAdministrador
PrecondiciónEl código debe estar en estado ISSUED y no haber vencido
Flujo principal
  1. El admin ingresa el código BDG-XXXXXX presentado por el miembro
  2. El sistema verifica que el código es válido y no está vencido
  3. El estado cambia a REDEEMED
  4. Se registra el admin que validó y la fecha de validación
  5. El admin entrega la recompensa al miembro
Flujo alternativo: Si el código ya fue usado o está vencido → error con detalle del estado actual
MÓDULO 9

Admin — Gestión de Promociones

CU-22 Crear una promoción
ActorAdministrador
Flujo principal
  1. Accede a "Promociones" en el panel admin
  2. Completa el formulario: título, descripción, tipo (PUBLIC o MEMBERS_ONLY), fechas de inicio y fin, tipo de descuento, valor e imagen
  3. La promoción se publica inmediatamente
TipoVisibilidad
PUBLICVisible para cualquier persona (con o sin sesión iniciada)
MEMBERS_ONLYVisible únicamente para miembros autenticados
